# Security Policy

## Supported Versions

Security fixes are prioritized for the latest state of the default branch.

| Version | Supported |
|---------|-----------|
| main    | :white_check_mark: |
| older snapshots | :x: |

## Reporting a Vulnerability

Please report vulnerabilities responsibly and avoid public disclosure until a
fix is available.

Preferred channel:

- GitHub private vulnerability reporting:
  https://github.com/kshidenko/openclaw-mem0-v2/security/advisories/new

If private reporting is unavailable, open a minimal issue (without exploit
details) and request a private contact channel:

- https://github.com/kshidenko/openclaw-mem0-v2/issues

## What to Include

- A clear description of the issue and impact
- Steps to reproduce or proof of concept
- Affected versions/commits
- Suggested mitigations (if known)

## Response Targets

- Initial triage: within 7 days
- Status update: within 14 days
- Fix timeline: depends on severity and complexity
