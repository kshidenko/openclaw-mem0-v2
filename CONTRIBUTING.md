# Contributing to OpenClaw Mem0 v2

Thanks for your interest in contributing.

## Ground Rules

- Be respectful and follow the `CODE_OF_CONDUCT.md`.
- Keep pull requests focused and small when possible.
- Prefer issues for design discussions before large changes.

## Ways to Contribute

- Report bugs and edge cases
- Suggest features or UX improvements
- Improve docs and examples
- Submit code fixes and tests

## Development Setup

1. Fork the repository and clone your fork.
2. Install dependencies:

```bash
npm install
```

3. Run your local checks before opening a pull request.

## Coding Guidelines

- Use clear, descriptive names.
- Keep functions focused and easy to test.
- Avoid breaking existing behavior without documenting it.
- Update docs when behavior or configuration changes.

## Commit and Pull Request Guidelines

- Use concise commit messages describing intent.
- Link relevant issues in your PR description.
- Include:
  - what changed
  - why it changed
  - how you validated it

If your change affects memory extraction/retrieval behavior, include a short
before/after example.

## Reporting Bugs

Please use the bug report template and include:

- expected behavior
- actual behavior
- reproduction steps
- environment details

## Security Issues

Do not open public issues for sensitive security vulnerabilities.
Follow `SECURITY.md` for responsible disclosure.
